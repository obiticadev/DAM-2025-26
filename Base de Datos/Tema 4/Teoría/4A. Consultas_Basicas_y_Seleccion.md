## 1. La sentencia SELECT

Para recuperar o seleccionar datos de una o varias tablas se utiliza la sentencia `SELECT`. Esta operaciﾃｳn se basa en dos conceptos relacionales:
*   **Proyecciﾃｳn:** Elegir quﾃｩ columnas queremos ver.
*   **Selecciﾃｳn:** Elegir quﾃｩ filas (registros) queremos ver basﾃ｡ndonos en criterios.

La sentencia consta de cuatro partes bﾃ｡sicas, siendo dos obligatorias y dos opcionales:

1. **Clﾃ｡usula SELECT** (Obligatoria): Indica quﾃｩ nombres de columnas queremos que se muestren.
2. **Clﾃ｡usula FROM** (Obligatoria): Indica de quﾃｩ tabla(s) se extraerﾃ｡n los datos.
3. **Clﾃ｡usula WHERE** (Opcional): Establece un criterio de selecciﾃｳn o condiciﾃｳn para filtrar los datos.
4. **Clﾃ｡usula ORDER BY** (Opcional): Establece el criterio de ordenaciﾃｳn.

### Sintaxis General
```sql
SELECT [ALL | DISTINCT] columna1, columna2, ...
FROM tabla1, tabla2, ...
WHERE condiciﾃｳn1, condiciﾃｳn2, ...
ORDER BY ordenaciﾃｳn;
```

### Opciones ALL y DISTINCT
*   **`ALL`**: Es el valor por defecto. Selecciona todas las filas, aunque estﾃｩn repetidas.
*   **`DISTINCT`**: Suprime aquellas filas del resultado que tengan igual valor que otras (elimina duplicados).

> **CASO DE USO PRﾃ，TICO (`DISTINCT`):** 
> *Problema:* Queremos saber en quﾃｩ ciudades tenemos clientes, pero un cliente puede vivir en la misma ciudad que otro, lo que nos darﾃｭa una lista llena de ciudades repetidas.
> *Soluciﾃｳn:* Usar `DISTINCT` para un listado limpio.
> ```sql
> SELECT DISTINCT ciudad FROM CLIENTES;
> ```

<a id="11-detalles-de-la-clausula-select"></a>
## 1.1. Detalles de la Clﾃ｡usula SELECT

Esta clﾃ｡usula define exactamente **quﾃｩ** queremos visualizar en el resultado.

*   **Nombrar columnas con su origen**: ﾃ嗾il para evitar ambigﾃｼedades.
    ```sql
    SELECT USUARIOS.Nombre, USUARIOS.Apellidos FROM USUARIOS;
    ```
*   **Seleccionar todas las columnas**: Usamos asterisco (`*`).
    ```sql
    SELECT * FROM EMPLEADOS;
    ```
*   **Alias de columnas**: Cambia el nombre de la cabecera. Si el alias contiene espacios o caracteres especiales, debe ir entre **comillas dobles**.
    ```sql
    -- Alias simple sin comillas
    SELECT SALARIO AS Sueldo FROM EMPLEADOS;
    
    -- Alias complejo con espacios
    SELECT F_Nacimiento "Fecha de Nacimiento" FROM USUARIOS;
    ```
*   **Uso de constantes, expresiones y funciones**:
    ```sql
    -- Inyectar una constante en todas las filas resultados
    SELECT Nombre, 'Activo' AS "Estado de Cuenta" FROM USUARIOS;
    ```

## 1.2. Detalles de la Clﾃ｡usula FROM

Define de **dﾃｳnde** se obtienen las columnas.

*   **Consultas combinadas (JOIN clﾃ｡sicos)**: Se separan por comas. (Se detalla en otro documento).
*   **Propietario de la tabla**: `ESQUEMA.TABLA`.

*   **Alias de tablas**: Abreviatura para simplificar consultas. Para asignarlo se deja un espacio (no `" "`).
    ```sql

    FROM USUARIOS u;
    ```

<a id="13-detalles-de-la-clausula-where"></a>
## 1.3. Detalles de la Clﾃ｡usula WHERE

Se utiliza para restringir la selecciﾃｳn a un subconjunto de filas que cumplan una condiciﾃｳn determinada. Va a continuaciﾃｳn de `WHERE`.

> **CASO DE USO PRﾃ，TICO (`WHERE` con textos e ignorando mayﾃｺsculas):**
> *Problema:* Buscar empleados que se llamen "Juan", pero en la BD podrﾃｭa estar guardado como "JUAN", "juan" o "Juan".
> *Soluciﾃｳn:* Convertir a mayﾃｺsculas al vuelo.
> ```sql
> SELECT nombre, apellidos FROM EMPLEADOS WHERE UPPER(nombre) = 'JUAN';
> ```

<a id="14-ordenacion-de-registros-clausula-order-by"></a>
## 1.4. Ordenaciﾃｳn de registros: Clﾃ｡usula ORDER BY

Permite especificar cﾃｳmo se ordenarﾃ｡ la respuesta de la consulta.

### Sintaxis
```sql
ORDER BY columna1 [ASC | DESC], columna2 [ASC | DESC];
```

### Reglas y Detalles
*   **Tipos de datos**: Se pueden ordenar caracteres, nﾃｺmeros o fechas.
*   **Direcciﾃｳn**: `ASC` (defecto) o `DESC`.
*   **Ordenaciﾃｳn mﾃｺltiple**:
    ```sql
    -- Ordena por Apellido de A a Z, y si hay dos iguales, ordena por Salario de mayor a menor.
    SELECT nombre, apellidos, salario 
    FROM EMPLEADOS 
    ORDER BY apellidos ASC, salario DESC;
    ```
*   **Ordenaciﾃｳn por posiciﾃｳn**: 
    ```sql
    SELECT nombre, apellidos, localidad 
    FROM usuarios 
    ORDER BY 3; -- Ordena por localidad
    ```

## 2. Operadores

### 2.1. Operadores de comparaciﾃｳn (Relacionales)

| Operador | Significado |
| :--- | :--- |
| `=`, `!=`, `< >`, `^=` | Igualdad / Desigualdad (Las 3 ﾃｺltimas son equivalentes) |
| `<`, `>`, `<=`, `>=` | Menor, Mayor, etc. |
| `IN` / `NOT IN` | Pertenece / No pertenece a una lista |
| `BETWEEN` / `NOT BETWEEN` | Dentro / Fuera de un rango numﾃｩrico o de fechas (inclusivo) |
| `LIKE` | Comparaciﾃｳn de patrones (`%` conjunto, `_` un carﾃ｡cter) |
| `IS NULL` / `IS NOT NULL`| Comprobaciﾃｳn de nulos |

*   **Valores `NULL`**: Nunca usar `=` con `NULL`. Siempre `IS NULL`. En `ORDER BY ASC`, los nulos van al final.

> **EJEMPLOS PRﾃ，TICOS DE OPERADORES RELACIONALES:**
> ```sql
> -- IN: Buscar usuarios de ciudades especﾃｭficas
> SELECT * FROM USUARIOS WHERE Localidad IN ('Madrid', 'Barcelona', 'Valencia');
> 
> -- BETWEEN: Filtrar por rango de fechas (el between INCLUYE los lﾃｭmites)
> SELECT * FROM EMPLEADOS WHERE Fecha_Alta BETWEEN '01/01/2023' AND '31/12/2023';
> 
> -- LIKE: Correo termina en @gmail.com
> SELECT Correo FROM CLIENTES WHERE Correo LIKE '%@gmail.com';
> 
> -- LIKE: Nombre de 4 letras que empieza por 'P'
> SELECT Nombre FROM ALUMNOS WHERE Nombre LIKE 'P___';
> ```

### 2.2. Operadores aritmﾃｩticos y de concatenaciﾃｳn

*   **Aritmﾃｩticos**: `+`, `-`, `*`, `/`. Si se opera sobre `NULL`, el resultado es `NULL`.
*   **Concatenaciﾃｳn (`||`)**: Une campos.

> **CASOS DE USO (Concatenaciﾃｳn y Cﾃ｡lculos):**
> ```sql
> -- Mostrar una ficha legible en una sola columna:
> SELECT 'El empleado ' || Nombre || ' ' || Apellidos || ' tiene un salario de ' || Salario AS "Ficha" FROM EMPLEADOS;
> 
> -- Aumento salarial del 10%
> SELECT Salario, Salario * 1.10 AS "Salario Aumentado" FROM EMPLEADOS;
> ```

<a id="23-operadores-logicos"></a>
### 2.3. Operadores Lﾃｳgicos

Permiten evaluar mﾃｺltiples condiciones conjuntas: `AND`, `OR`, `NOT`.

> **EJEMPLOS DE COMPLEJIDAD Lﾃ敵ICA:**
> ```sql
> -- Empleados del departamento 10 con salario mayor a 2000
> SELECT * FROM EMPLEADOS WHERE Dept_no = 10 AND Salario > 2000;
> 
> -- Empleados directivos (dep 10) o del departamento informﾃ｡tico (dep 20)
> SELECT * FROM EMPLEADOS WHERE Dept_no = 10 OR Dept_no = 20; 
> -- (Equivalente a: WHERE Dept_no IN (10, 20))
> 
> -- Empleados que NO son de Madrid
> SELECT * FROM EMPLEADOS WHERE NOT Localidad = 'Madrid';
> ```

<a id="24-precedencia-de-operadores"></a>
### 2.4. Precedencia de Operadores

Prioridad (de mayor a menor):
1.  Parﾃｩntesis (Uso indispensable para evitar errores lﾃｳgicos).
2.  `*`, `/`
3.  `+`, `-`
4.  `||`
5.  Relacionales (`=`, `<`, `LIKE`, `IN`...)
6.  `NOT`
7.  `AND`
8.  `OR`

> **CASO DE USO (Precedencia):**
> *Problema:* Queremos administradores que ganen mﾃ｡s de 2000 o usuarios de soporte que ganen mﾃ｡s de 2000.
> *Mala consulta:* `WHERE Rol = 'Admin' OR Rol = 'Soporte' AND Salario > 2000` (El AND se ejecuta antes, darﾃｭa soporte con >2000 y TODOS los Admins, sean del salario que sean).
> *Buena consulta:*
> ```sql
> SELECT * FROM EMPLEADOS WHERE (Rol = 'Admin' OR Rol = 'Soporte') AND Salario > 2000;
> ```
