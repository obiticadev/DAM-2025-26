# 📚 Cheatsheet: Lenguaje XML

## 1. ¿Qué es XML?

**XML** (eXtensible Markup Language) es un subconjunto de **SGML**, simplificado y adaptado a Internet. 
*   **Legibilidad:** Es legible y comprensible para cualquier usuario aunque no tenga conocimientos de lenguajes de marcas, ya que las etiquetas aportan el significado al contenido que encierran.
*   **Metalenguaje:** No es un lenguaje de marcas propiamente dicho, sino un **metalenguaje**. Esto significa que permite definir las reglas que debe cumplir un lenguaje de marcas para un uso determinado (te permite inventar tus propias etiquetas).
*   **Origen:** Fue creado por el **W3C** (World Wide Web Consortium) a finales de los años 90, publicándose su primera versión en **1998**.
*   **Finalidad:** Permite representar **información estructurada** en la web de modo que pueda ser almacenada, transmitida, procesada, visualizada e impresa por diversos tipos de aplicaciones y dispositivos.

---

## 2. Usos Principales de XML

El uso fundamental de XML se basa en el **almacenamiento y distribución de información**:

1.  **Intercambio de información entre aplicaciones:** No requiere un software especial para leer su contenido, dado que la información se almacena en **texto plano**.
2.  **Computación distribuida:** Consiste en intercambiar información entre diferentes computadoras a través de la red. La gran ventaja es que los documentos XML son **inocuos**: no representan riesgos de seguridad ya que no pueden contener código maligno ni virus.
3.  **Información empresarial:** Se utiliza para generar documentos empresariales gracias a su gran facilidad para estructurar los datos del modo más adecuado para cada empresa.

## 3. Estructura de un documento XML

Un documento XML consiste en un fichero de texto plano (con extensión **`.xml`**) que contiene un conjunto de datos delimitados por sus correspondientes etiquetas de marcado.

### 3.1. Estructura Física (Bloques del documento)
Se distinguen dos bloques claramente diferenciados en el archivo:
1.  **El Prólogo (o cabecera):** Contiene información sobre la definición, versión y codificación del documento.
2.  **El Cuerpo:** Contiene los datos reales y las etiquetas de marcado.

**Ejemplo de estructura:**
```xml
<!-- PROLOGO -->
<?xml version="1.0" encoding="ISO-8859-1" standalone="no"?>
<!DOCTYPE persona SYSTEM "persona.dtd">

<!-- CUERPO -->
<persona>
    <nombre>Luis</nombre>
    <apellidos>Pérez</apellidos>
</persona>
```

### 3.2. Estructura Lógica (Árbol Dirigido)
Lógicamente, un documento XML tiene la estructura de un **árbol dirigido**, donde cada "nodo" representa un elemento del documento. Todos los nodos, excepto uno, tienen un único padre.
*   **Nodo raíz:** Es el nodo principal que agrupa a todos los demás y **no tiene padre** (en el ejemplo anterior, sería `<persona>`).
*   **Nodos hermanos:** Son aquellos nodos que comparten el mismo padre (ej: `<nombre>` y `<apellidos>`).
*   **Nodos descendientes:** Todos los nodos a los que se llega bajando desde un nodo concreto (sus hijos, los hijos de sus hijos, etc.).
*   **Nodos ascendientes:** Todos los nodos de los que un nodo es descendiente yendo hacia arriba (su padre, el abuelo, etc.).

## 4. Componentes de un documento XML

### 4.1. Etiquetas (Tags)
Son las marcas que sirven para identificar un contenido concreto y separarlo del resto del documento. Se encierran entre corchetes angulares `< >`. Se clasifican en tres tipos:
*   **Etiquetas de apertura (start-tag):** Indican dónde empieza el contenido.
    ```xml
    <apartado>
    ```
*   **Etiquetas de cierre (end-tag):** Indican dónde termina el contenido. Siempre empiezan con una barra diagonal `/`.
    ```xml
    </apartado>
    ```
*   **Etiquetas vacías (empty-tag):** Etiquetas que no encierran ningún contenido. Terminan con la barra diagonal `/` al final.
    ```xml
    <salto-de-linea />
    ```

### 4.2. Elementos
Son los bloques estructurales que conforman el documento. Un elemento comienza con una etiqueta de apertura y termina con una de cierre (encerrando texto u otros elementos), o bien consiste en una única etiqueta vacía. 
Los elementos **pueden contener atributos**.

*   **Elemento con texto:**
    ```xml
    <autor>Maria Jimenez</autor>
    ```
*   **Elemento que contiene otros elementos:**
    ```xml
    <web>
        <autor>Maria Jimenez</autor>
    </web>
    ```
*   **Elemento vacío (dos formas válidas):**
    ```xml
    <ingreso /> 
    <!-- o bien -->
    <ingreso></ingreso>
    ```

### 4.3. Atributos
Son propiedades que se añaden a las etiquetas para proporcionar información adicional. Consisten en una pareja **`nombre="valor"`**.
*   **Regla vital 1:** Solo pueden aparecer en las **etiquetas de apertura** y en las **etiquetas vacías**. ¡NUNCA en las de cierre!
*   **Regla vital 2:** En una misma etiqueta **no pueden existir dos atributos con el mismo nombre**.

**Ejemplo de Atributo:**
```xml
<!-- 'pais' es el atributo, '"España"' es el valor del atributo -->
<autor pais="España">Jose Ramón</autor>
```

### 4.4. Instrucciones de Procesamiento
Son etiquetas especiales dirigidas a las aplicaciones que leen y procesan el documento. 
*   Se delimitan por **`<?`** y **`?>`**.
*   Normalmente se sitúan al principio del documento.

**Ejemplo para declarar el documento XML:**
```xml
<?xml version="1.0" encoding="utf-8"?>
```
**Ejemplo para asociar una hoja de estilos (XSL):**
```xml
<?xml-stylesheet type="text/xsl" href="estilo.xsl" ?>
```

### 4.5. Entidades y Referencias a entidades
*   **Entidades:** Son similares a las "constantes" en programación. Se definen para almacenar un valor que se va a repetir y se definen con la instrucción `<!ENTITY ...>`.
    ```xml
    <!ENTITY web "Ejemplo">
    ```
*   **Referencias a entidades:** Se utilizan para invocar el valor de una entidad previamente definida en el cuerpo del documento. Empiezan con el carácter ampersand (**`&`**), le sigue el nombre, y terminan con un punto y coma (**`;`**)*. Al procesar el XML, el programa sustituye la referencia por su valor.
    ```xml
    <!-- El XML escrito -->
    <autor>&web;</autor>
    
    <!-- El procesador XML lo convertirá a: -->
    <autor>Ejemplo</autor>
    ```
    *(Nota de sintaxis: Aunque el texto de la presentación indica erróneamente que "terminan con un punto", el propio ejemplo de la diapositiva muestra correctamente el uso del punto y coma `;`).*

### 4.6. Secciones CDATA (Character Data)
Son áreas de texto cuyo contenido el procesador XML debe **interpretar puramente como texto y no como marcas de código**. Es muy útil si necesitas escribir ejemplos de código HTML/XML dentro de tu archivo sin que el procesador intente ejecutarlos o interpretarlos.
*   Se delimitan por **`<![CDATA[`** y **`]]>`**.

**Ejemplo CDATA:**
```xml
<ejemplo>
    <![CDATA[
        En HTML la negrita se escribe: <strong>texto</strong>
    ]]>
</ejemplo>
```

## 5. Ejemplo Completo y Nomenclatura del Árbol XML

### Documento XML (`biblioteca.xml`)
```xml
<?xml version="1.0" encoding="UTF-8"?>
<biblioteca>
    <libro>
        <titulo>La vida está en otra parte</titulo>
        <autor>Milan Kundera</autor>
        <fechaPublicacion año="1973"/>
    </libro>
    <libro>
        <titulo>Pantaleón y las visitadoras</titulo>
        <autor fechaNacimiento="28/03/1936">Mario Vargas Llosa</autor>
        <fechaPublicacion año="1973"/>
    </libro>
    <libro>
        <titulo>Conversación en la catedral</titulo>
        <autor fechaNacimiento="28/03/1936">Mario Vargas Llosa</autor>
        <fechaPublicacion año="1969"/>
    </libro>
</biblioteca>
```

### Clasificación de los Nodos según su representación en Árbol
Si dibujamos el árbol del documento anterior, identificaríamos los siguientes tipos de nodos:
1.  **Nodo Raíz:** El origen del documento, representado como `/`, y del cual cuelga el elemento principal `<biblioteca>`.
2.  **Nodo Elemento:** Cualquier etiqueta estructural del árbol. Ejemplos: `<libro>`, `<titulo>`, `<autor>`, `<fechaPublicacion>`.
3.  **Nodo Atributo:** Propiedades anidadas en los elementos. Ejemplos: `año="1973"`, `fechaNacimiento="28/03/1936"`.
4.  **Nodo de Texto:** La información pura y dura alojada dentro de los elementos. Ejemplos: `"La vida está en otra parte"`, `"Milan Kundera"`, `"Mario Vargas Llosa"`.
