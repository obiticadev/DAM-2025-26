# 📚 Cheatsheet: Lenguaje XML - Espacios de Nombres (Namespaces)

## 1. ¿Qué son los espacios de nombres? (El Problema y la Solución)

**El Problema (Colisión de etiquetas):** 
Puede ocurrir que una aplicación que procesa documentos XML necesite manejar o combinar varios ficheros que utilicen **exactamente las mismas etiquetas** para propósitos distintos. Esto representa un problema grave, ya que el procesador no sabría identificar a qué gramática (o esquema) pertenece cada etiqueta.

**La Solución:** 
Para resolver esta ambigüedad se utilizan los **espacios de nombres** (o *namespacing*). Consiste en incluir en cada etiqueta un código (prefijo) que indica de forma inequívoca el esquema o gramática al que pertenece.

---

## 2. El Atributo `xmlns` y la asignación de prefijos

Para declarar un espacio de nombres se utiliza el atributo especial **`xmlns`** (XML Namespace). Este atributo permite asignar un espacio de nombres (habitualmente una URL/URI identificativa) a un prefijo que luego se añadirá delante de cada etiqueta.

### Ejemplo del Problema (Sin espacios de nombres)
Imagina un documento donde la etiqueta `<title>` se repite en dos contextos totalmente distintos:
1.  Para indicar el título genérico del documento XML.
2.  Para indicar el título de una página web en HTML incrustada.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<document>
    <!-- Título genérico del documento -->
    <title>Documento de prueba</title> 
    <content>
        <html>
            <head>
                <!-- Título HTML -->
                <title>Titulo HTML</title>
            </head>
            <body>
                Texto del documento
            </body>
        </html>
    </content>
    <author>Jorge</author>
</document>
```
*En el código anterior, el procesador XML se confundiría al ver dos etiquetas `<title>` con propósitos diferentes.*

### Solución con Prefijos Explícitos
La mejor solución es usar el atributo `xmlns:prefijo="URL"` en el elemento raíz para definir los diccionarios, y luego anteponer ese `prefijo:` a cada etiqueta correspondiente.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!-- Se declaran DOS espacios de nombres con sus respectivos prefijos: 'prefix' y 'html' -->
<document xmlns:prefix="http://www.dominio.com/esquema.xsd" 
          xmlns:html="http://www.w3c.org/html">
    
    <!-- Etiquetas pertenecientes al espacio 'prefix' -->
    <prefix:title>Documento de prueba</prefix:title>
    <prefix:content>
        
        <!-- Etiquetas pertenecientes al espacio 'html' -->
        <html:html>
            <html:head>
                <html:title>Titulo HTML</html:title>
            </html:head>
            <html:body>
                Texto del documento
            </html:body>
        </html:html>
        
    </prefix:content>
    <prefix:author>Jorge</prefix:author>
</document>
```

## 3. Espacio de Nombres por Defecto (Default Namespace)

En muchas ocasiones, la inmensa mayoría de las etiquetas de un documento pertenecen a un mismo espacio de nombres principal, y solo unas pocas pertenecen a un vocabulario externo.

Para no tener que escribir el prefijo en absolutamente todas las etiquetas del documento, XML permite definir un **espacio de nombres por defecto**. 

**¿Cómo funciona?**
Se declara usando el atributo `xmlns="URL"` **(sin los dos puntos ni el nombre del prefijo)**. Al hacer esto, **no obliga** a indicar ningún prefijo para las etiquetas que pertenezcan a ese esquema principal; cualquier etiqueta sin prefijo pertenecerá automáticamente a él.

### Ejemplo con Espacio de Nombres por Defecto

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!-- El espacio principal NO tiene prefijo (espacio por defecto). El espacio secundario sí tiene prefijo (html) -->
<document xmlns="http://www.dominio.com/esquema.xsd" 
          xmlns:html="http://www.w3c.org/html">
    
    <!-- Estas etiquetas asumen automáticamente el espacio por defecto (no llevan prefijo) -->
    <title>Documento de prueba</title>
    <content>
        
        <!-- Estas etiquetas usan su prefijo específico 'html' para diferenciarse -->
        <html:html>
            <html:head>
                <html:title>Titulo HTML</html:title>
            </html:head>
            <html:body>
                Texto del documento
            </html:body>
        </html:html>
        
    </content>
    <author>Jorge</author>
</document>
```
