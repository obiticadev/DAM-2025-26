# 📚 Cheatsheet: Lenguaje XML - Esquemas XSD

## 1. ¿Qué es XSD (XML Schema Definition)?

Un **esquema XSD** es un documento que define la estructura válida de un documento XML. Establece qué elementos, atributos y entidades pueden aparecer, su orden, cardinalidad (número de repeticiones permitidas) y la relación jerárquica (quiénes son hijos de quiénes).

Es el mecanismo moderno que sustituye a los antiguos DTD y ofrece **ventajas cruciales**:
1.  **Es un documento XML en sí mismo:** A diferencia de DTD (que es texto plano), XSD usa la sintaxis XML, por lo que podemos comprobar si él mismo está "bien formado".
2.  **Tipado de datos:** Posee una extensa lista de tipos de datos predefinidos (números, fechas, cadenas, etc.) que, además, puede ser ampliada por el usuario.
3.  **Cardinalidad precisa:** Permite definir exactamente cuántas veces puede repetirse un elemento (ej: mínimo 2, máximo 5), algo muy limitado en DTD.
4.  **Soporta Espacios de Nombres (Namespaces):** Permite mezclar distintos vocabularios sin que haya colisión de etiquetas.

## 2. Estructura y Referencia de un Esquema XSD

### 2.1. Estructura interna del archivo XSD (extensión `.xsd`)
El esquema XSD utiliza el espacio de nombres oficial del W3C para definir sus etiquetas.
*   `xs:schema`: Es la **etiqueta raíz**. Define el espacio de nombres del esquema.
*   `xs:element`: Define cuál será el elemento raíz del documento XML que validaremos.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">
    <xs:element name="alumnos" />
</xs:schema>
```

### 2.2. Referencia al esquema desde el archivo XML
Para que un documento XML se valide contra un XSD, debe referenciarlo en su etiqueta raíz mediante dos atributos:
*   `xmlns:xsi`: Declara el espacio de nombres para instanciar el esquema.
*   `xsi:noNamespaceSchemaLocation`: Vincula el XML con el archivo `.xsd` local.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<alumnos xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:noNamespaceSchemaLocation="alumnos.xsd">
    ...
</alumnos>
```

## 3. Declaraciones de Elementos (`xs:element`)

Permiten declarar qué etiquetas XML están permitidas.

**Sintaxis base:**
```xml
<xs:element name="nombre" type="xs:string" default="Ejemplo" minOccurs="1" maxOccurs="unbounded" />
```

**Atributos de `xs:element`:**
*   `name`: Indica el nombre del elemento. **Obligatorio** si el elemento cuelga directamente de la raíz `xs:schema`.
*   `ref`: Indica que la declaración de este elemento se encuentra en otro lugar del esquema. **No permitido** si el padre es `xs:schema`. *Ojo: Nunca puede aparecer junto con el atributo `name` ni con `type`*.
*   `type`: Indica el tipo de dato que almacenará (ej: `xs:string`, `xs:integer`).
*   `default`: Valor por defecto si la etiqueta aparece en el XML vacía (sin valor).
*   `fixed`: Establece un valor único e inamovible para ese elemento.
*   `minOccurs` / `maxOccurs`: Indican el mínimo y máximo número de veces que el elemento puede aparecer. El valor por defecto para ambos es **`1`**. (Para infinito se usa `"unbounded"`). *No permitidos si el padre directo es `xs:schema`.*

---

## 4. Declaraciones de Atributos (`xs:attribute`)

Permiten declarar qué atributos pueden acompañar a las etiquetas.

**Sintaxis base:**
```xml
<xs:attribute name="moneda" type="xs:string" default="euro" use="required" />
```

**Atributos de `xs:attribute`:**
*   `name`: Nombre del atributo.
*   `ref`: Hace referencia a un atributo definido en otro lugar. (No puede ir con `name` ni con `type`).
*   `type`: Tipo de dato numérico, texto, fecha, etc.
*   `use`: Indica el grado de obligatoriedad del atributo. Sus valores posibles son:
    *   `optional` (Es el valor **por defecto**).
    *   `required` (Obligatorio).
    *   `prohibited` (Prohibido en este contexto).
*   `default`: Valor por defecto si el atributo no se escribe en el documento XML.
*   `fixed`: Único valor permitido para el atributo.

## 5. Tipos de Datos

XSD diferencia los datos en dos grandes bloques: Simples y Complejos.

### 5.1. Tipos de Datos Simples (`xs:simpleType`)
Contienen únicamente valores (texto, números) y **no pueden contener ni atributos ni otros elementos hijos**.
Se dividen en:
1.  **Predefinidos:** Hay 44 tipos ordenados jerárquicamente. El tipo base absoluto es `xs:anyType`. De `xs:anySimpleType` cuelgan todos los primitivos (`string`, `boolean`, `dateTime`, `time`, `date`, `float`, `decimal`, `double`...). De estos cuelgan los derivados (ej: de `decimal` cuelga `integer`, y de este `long`, `int`, `positiveInteger`, etc.).
2.  **Definidos por el usuario:** Se crean partiendo de un tipo predefinido y aplicándole **restricciones**.

**Forma 1: Definición directa (anidada) en el elemento:**
```xml
<xs:element name="edad" minOccurs="1" maxOccurs="1">
    <xs:simpleType>
        <xs:restriction base="xs:integer">
            <xs:minInclusive value="0"/>
            <xs:maxInclusive value="100"/>
        </xs:restriction>
    </xs:simpleType>
</xs:element>
```

**Forma 2: Definición con nombre (reutilizable):**
```xml
<!-- Se define el tipo globalmente -->
<xs:simpleType name="longitudMaxima">
    <xs:restriction base="xs:string">
        <xs:minLength value="0"/>
        <xs:maxLength value="10"/>
    </xs:restriction>
</xs:simpleType>

<!-- Se usa llamándolo por su nombre en el atributo 'type' -->
<xs:element name="nombre" type="longitudMaxima"/>
```

### 5.2. Tipos de Datos Complejos (`xs:complexType`)
Se utilizan para elementos que **contienen atributos**, **contienen otros elementos hijos**, o son mixtos/vacíos.
*   `xs:simpleContent`: El elemento solo contiene texto, pero tiene atributos.
*   `xs:complexContent`: El elemento contiene otros elementos descendientes (y opcionalmente texto, lo que se llama contenido mixto).

**Ejemplo de creación de tipo complejo:**
```xml
<xs:complexType name="infoPersona">
    <xs:sequence>
        <xs:element name="nombre" type="xs:string"/>
        <xs:element name="edad" type="xs:integer"/>
    </xs:sequence>
    <!-- Los atributos siempre se declaran al final del complexType -->
    <xs:attribute name="numero" type="xs:integer"/>
</xs:complexType>
```

**Ejemplo de extensión de un tipo complejo (`xs:extension`):**
```xml
<xs:complexType name="infoPersonaAmpliada">
    <xs:complexContent>
        <xs:extension base="infoPersona">
            <xs:sequence>
                <xs:element name="ciudad" type="xs:string"/>
                <xs:element name="pais" type="xs:string"/>
            </xs:sequence>
        </xs:extension>
    </xs:complexContent>
</xs:complexType>

<xs:element name="persona" type="infoPersonaAmpliada" />
```

## 6. Indicadores en Tipos Complejos

Los indicadores agrupan y organizan a los elementos hijos dentro de un `xs:complexType`.

### A. Indicadores de Orden
Dictan cómo deben aparecer los elementos descendientes:
*   **`xs:sequence`**: Define que los elementos deben aparecer **exactamente en el mismo orden** en el que se declaran.
*   **`xs:choice`**: Es una alternativa. De toda la lista de elementos anidados, **solo puede aparecer uno**.
*   **`xs:all`**: Permite que aparezcan todos los elementos declarados, pero en **cualquier orden**.

### B. Indicadores de Ocurrencia
*   `minOccurs` y `maxOccurs`: Indican cuántas veces se puede repetir el propio bloque agrupador (ej: repetir una `xs:choice` entera).

### C. Indicadores de Grupo
*   **`xs:group`**: Permite definir un bloque de elementos con un nombre, para luego insertarlo (con el atributo `ref`) dentro de otros tipos complejos, favoreciendo la reutilización.
*   **`xs:attributeGroup`**: Igual que el anterior, pero agrupando atributos.

**Ejemplo integrador de Indicadores y Grupos:**
```xml
<!-- 1. Definición del grupo reutilizable -->
<xs:group name="direccion">
    <xs:sequence>
        <xs:element name="calle" type="xs:string"/>
        <xs:element name="localidad" type="xs:string"/>
        <xs:element name="provincia" type="xs:string"/>
    </xs:sequence>
</xs:group>

<!-- 2. Uso del grupo e indicadores de orden -->
<xs:element name="alumno" minOccurs="1" maxOccurs="1">
    <xs:complexType>
        <xs:sequence>
            <xs:element name="nif" type="xs:string" />
            
            <xs:choice minOccurs="1" maxOccurs="1">
                <xs:element name="nombre" type="xs:string" minOccurs="1" maxOccurs="1" />
                <xs:element name="nombre_compuesto" type="xs:string" minOccurs="1" maxOccurs="1" />
            </xs:choice>
            
            <xs:element name="apellido" type="xs:string" minOccurs="2" maxOccurs="2" />
            
            <!-- Inserción del grupo 'direccion' -->
            <xs:group ref="direccion" minOccurs="1" maxOccurs="1" />
        </xs:sequence>
    </xs:complexType>
</xs:element>
```

## 7. Restricciones (Facetas)

Las restricciones se aplican dentro de un bloque `<xs:restriction base="...">` (habitualmente en un `xs:simpleType`) para limitar los valores permitidos de un dato.

### 7.1. Rango de Números
Para datos numéricos o fechas/horas:
*   `minInclusive` / `maxInclusive`: Incluyen el valor límite.
*   `minExclusive` / `maxExclusive`: Excluyen el valor límite.
```xml
<!-- Inclusivo (Entre 0 y 100, ambos incluidos) -->
<xs:restriction base="xs:integer">
    <xs:minInclusive value="0"/>
    <xs:maxInclusive value="100"/>
</xs:restriction>

<!-- Exclusivo (Mayor a 0 y menor a 100) -->
<xs:restriction base="xs:integer">
    <xs:minExclusive value="0"/>
    <xs:maxExclusive value="100"/>
</xs:restriction>
```

### 7.2. Dígitos (`totalDigits`)
Define el número máximo de dígitos totales en números.
```xml
<xs:restriction base="xs:integer">
    <xs:totalDigits value="9"/>
</xs:restriction>
```

### 7.3. Lista de Valores (`enumeration`)
Restringe el elemento a uno de los valores de una lista predefinida.
```xml
<xs:restriction base="xs:string">
    <xs:enumeration value="Audi"/>
    <xs:enumeration value="Golf"/>
    <xs:enumeration value="BMW"/>
    <xs:enumeration value=""/>
</xs:restriction>
```

### 7.4. Longitud de Cadenas
Limita la longitud en caracteres de un tipo texto:
*   `length`: Longitud exacta obligatoria.
*   `minLength` / `maxLength`: Longitud mínima y máxima.
```xml
<!-- Longitud exacta de 5 -->
<xs:restriction base="xs:string">
    <xs:length value="5"/>
</xs:restriction>

<!-- Longitud entre 5 y 8 -->
<xs:restriction base="xs:string">
    <xs:minLength value="5"/>
    <xs:maxLength value="8"/>
</xs:restriction>
```

### 7.5. Expresión Regular (`pattern`)
Obliga a que el texto cumpla con un patrón de caracteres (Regex).
```xml
<!-- Obliga a 3 caracteres alfabéticos, el primero en mayúscula -->
<xs:restriction base="xs:string">
    <xs:pattern value="[A-Z][A-Za-z][A-Za-z]"/>
</xs:restriction>
```

### 7.6. Tratamiento de Espacios en Blanco (`whiteSpace`)
Define cómo tratar saltos de línea, espacios y tabuladores:
*   `preserve`: Respeta y mantiene los espacios.
*   `replace`: Reemplaza tabuladores/saltos de línea por espacios normales.
*   `collapse`: Reduce múltiples espacios a uno solo y quita los de los bordes.
```xml
<xs:restriction base="xs:string">
    <xs:whiteSpace value="preserve"/>
</xs:restriction>
```
