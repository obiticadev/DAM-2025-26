# 📚 Cheatsheet: Lenguaje XML - Validación de Documentos

## 1. ¿Qué es la validación de documentos?

La validación de documentos XML es un proceso de verificación doble. Consiste en comprobar que el documento cumple dos grandes requisitos:
1.  Que está **bien formado**: Es decir, que cumple a rajatabla todas las reglas sintácticas del lenguaje XML.
2.  Que es **válido**: Es decir, que cumple con las características, estructura y gramática que le impone el **esquema XML** que tiene asociado.

---

## 2. Reglas Sintácticas (Documento "Bien Formado")

Si un documento no está "bien formado", **no se considera un documento XML**. Por lo tanto, las aplicaciones o navegadores que procesan documentos XML están obligados a rechazar aquellos que no cumplan con las siguientes reglas sintácticas fundamentales:

### Reglas Generales y de Caracteres
*   **Caracteres válidos:** El documento debe contener únicamente caracteres válidos.
*   **Uso reservado de `<` y `&`:** Los caracteres menor que (`<`) y ampersand (`&`) **solo** se pueden utilizar como comienzo de etiquetas (en el caso de `<`) o comienzo de referencias a entidades (en el caso de `&`).

### Reglas de la Estructura (Árbol)
*   **Elemento raíz único:** Debe existir un **único elemento raíz** que contenga al resto de elementos del documento, formando así la estructura lógica de árbol.
*   **Anidamiento correcto:** Las etiquetas de apertura, las etiquetas de cierre y las etiquetas vacías deben estar **correctamente anidadas** (si abres la etiqueta A y luego la B, debes cerrar primero la B y luego la A: `<a><b>...</b></a>`).

### Reglas para los Nombres (Etiquetas y Atributos)
Los nombres elegidos para crear las etiquetas y los atributos deben seguir una nomenclatura estricta:
*   Deben comenzar siempre por una **letra**.
*   A esta primera letra le pueden seguir más letras, números, guiones altos (`-`), guiones bajos (`_`) y puntos (`.`).
*   **Prohibición estricta:** Los nombres **nunca contendrán espacios** en blanco.

### Reglas Específicas para Etiquetas
*   **Case-sensitive (Sensibilidad a mayúsculas):** Las etiquetas de cierre tienen que coincidir exactamente con las de apertura, respetando el uso de mayúsculas y minúsculas (Ej: `<Nombre>` debe cerrarse con `</Nombre>`, no vale `</nombre>`).
*   **Sin atributos en el cierre:** Las etiquetas de cierre **nunca** contienen atributos.

### Reglas Específicas para Atributos
*   El nombre de los atributos sigue exactamente las **mismas reglas** que el nombre de las etiquetas (empiezan por letra, sin espacios, etc.).
*   **Unicidad:** Ninguna etiqueta puede tener dos atributos con el mismo nombre.
*   **Obligatoriedad de valor:** Todos los atributos tienen que tener **algún valor** asignado (no existen atributos vacíos sin un `=`).
*   **Uso de comillas:** Los valores de los atributos deben estar obligatoriamente encerrados entre **comillas dobles (`" "`) o simples (`' '`)**.
*   **Sin referencias:** No pueden existir referencias a entidades dentro de los valores de los atributos.

## 3. ¿Qué es un esquema XML? (Documento "Válido")

Un **esquema XML** es un documento XML en sí mismo que define la **gramática** exacta de lo que está permitido en otro documento XML que haga uso de dicho esquema.

Esta gramática establece:
1.  Qué elementos y atributos concretos pueden aparecer.
2.  La secuencia (el orden) en el que deben aparecer los elementos.
3.  El número de repeticiones permitidas para cada elemento.
4.  El tipo de contenido (texto, número, fecha...) que puede albergar cada uno de dichos elementos.

### Requisitos para que un documento sea "Válido"
Por lo tanto, para que un documento que ya está *bien formado* se considere **válido**, debe cumplir estas tres condiciones:
1.  Incluir una **referencia** que apunte a un esquema.
2.  Incluir **únicamente** elementos y atributos que estén definidos previamente en dicho esquema (no puedes inventarte etiquetas que el esquema no conozca).
3.  Cumplir a rajatabla todas las **reglas gramaticales** (orden, tipo de datos, repeticiones) definidas en el esquema.

---

## 4. Definición de Esquemas XML (Modelos)

Para definir las reglas y gramáticas de los esquemas XML, la industria utiliza principalmente dos modelos:

1.  **DTD (Document Type Definition):** 
    *   Es el modelo más antiguo.
    *   Fue heredado directamente del lenguaje SGML.
2.  **XSD (XML Schema):** 
    *   Es el modelo más moderno y potente.
    *   Fue creado por el **W3C** (World Wide Web Consortium) para actuar como el sucesor natural y definitivo de DTD.
