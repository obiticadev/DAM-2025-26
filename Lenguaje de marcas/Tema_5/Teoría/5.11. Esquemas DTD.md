# 📚 Cheatsheet: Lenguaje XML - Esquemas DTD

## 1. ¿Qué es DTD?
**DTD** (*Document Type Definition*) es un documento de texto plano que define la estructura válida de un documento XML. 
*   Establece las reglas del documento: qué elementos, atributos y entidades pueden aparecer, en qué orden, cuántas veces pueden repetirse y cuáles pueden ser hijos de cuáles.
*   El procesador XML lo utiliza para verificar que un documento es **válido**.
*   Fue el **primer mecanismo de validación** utilizado para XML (heredado de SGML), siendo sustituido posteriormente por los esquemas XSD (*XML Schema*).

---

## 2. Referencia a un esquema DTD
El esquema DTD es un documento externo al documento XML (tiene extensión **`.dtd`**). Para que el XML se valide contra él, debe referenciarse en la cabecera del documento XML mediante la etiqueta `<!DOCTYPE>`.

**Sintaxis:**
```xml
<!DOCTYPE nodo_raíz SYSTEM "fichero.dtd">
```
**Ejemplo en un documento XML:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE agenda SYSTEM "agenda.dtd">
<agenda>
    ...
</agenda>
```

---

## 3. Tipos de Declaraciones en DTD
Los esquemas DTD describen la estructura mediante tres tipos principales de declaraciones:
1.  **Declaraciones de elementos:** Indican los elementos (etiquetas) permitidos y su contenido.
2.  **Declaraciones de entidades:** Indican las constantes definidas para el documento.
3.  **Declaraciones de atributos:** Indican los atributos permitidos para cada elemento y el tipo de los valores permitidos.

---

## 4. Declaraciones de Elementos
Definen de qué está compuesto cada elemento del XML.

**Sintaxis general:**
```xml
<!ELEMENT nombreElemento contenido >
```

### Tipos de Contenido Básicos:
*   **`EMPTY`**: El elemento es vacío (no puede tener contenido, ni texto ni otros elementos).
    *   *DTD:* `<!ELEMENT ejemplo EMPTY>`
    *   *XML Válido:* `<ejemplo></ejemplo>` o `<ejemplo />`
*   **`(#PCDATA)`**: *Parsed Character Data*. Indica que el elemento solo puede contener texto, pero NO otros elementos hijos.
    *   *DTD:* `<!ELEMENT ejemplo (#PCDATA)>`
    *   *XML Válido:* `<ejemplo />` o `<ejemplo>Esto es un ejemplo</ejemplo>`
*   **`ANY`**: El elemento puede contener cualquier cosa (texto libre y/u otros elementos definidos).
    *   *DTD:* `<!ELEMENT ejemplo ANY>` y `<!ELEMENT a ANY>`
    *   *XML Válido:* `<ejemplo />`, `<ejemplo>Texto</ejemplo>` o `<ejemplo><a>Texto</a></ejemplo>`

---

## 5. Declaraciones de Descendientes (Anidamiento y Cardinalidad)
Si un elemento contiene otros elementos (hijos), estos se describen **entre paréntesis**, utilizando expresiones para definir su orden y número de repeticiones (cardinalidad).

### A. Operadores de Secuencia y Selección
*   **Coma ( `,` ) - Secuencia Estricta:**
    Los elementos deben aparecer exactamente en el orden declarado. Todos son obligatorios.
    *   *DTD:* `<!ELEMENT ejemplo (a, b)>`
    *   *XML Válido:* `<ejemplo><a /><b /></ejemplo>`
*   **Pipe ( `|` ) - Opción Excluyente:**
    Aparece uno u otro, pero **nunca más de uno** a la vez.
    *   *DTD:* `<!ELEMENT ejemplo (a | b)>`
    *   *XML Válido:* `<ejemplo><a /></ejemplo>` o `<ejemplo><b /></ejemplo>`

### B. Operadores de Cardinalidad (Repeticiones)
*   **Interrogante ( `?` ) - Opcional:**
    Puede aparecer cero o una vez (aparece o no aparece).
    *   *DTD:* `<!ELEMENT ejemplo (a, b?)>`
    *   *XML Válido:* `<ejemplo><a /></ejemplo>` o `<ejemplo><a /><b /></ejemplo>`
*   **Asterisco ( `*` ) - Múltiple Opcional:**
    Puede no aparecer, o aparecer una o múltiples veces (cero o más).
    *   *DTD:* `<!ELEMENT ejemplo (a*, b)>`
    *   *XML Válido:* `<ejemplo><b /></ejemplo>` o `<ejemplo><a /><a /><b /></ejemplo>`
*   **Suma ( `+` ) - Múltiple Obligatorio:**
    Tiene que aparecer obligatoriamente **una o más veces**.
    *   *DTD:* `<!ELEMENT ejemplo (a+, b)>`
    *   *XML Válido:* `<ejemplo><a /><b /></ejemplo>` o `<ejemplo><a /><a /><b /></ejemplo>`

### C. Agrupación
*   **Paréntesis `( )`**: Se utilizan para agrupar expresiones lógicas más complejas.
    *   *DTD:* `<!ELEMENT ejemplo (a, (a|b))>` *(Debe llevar 'a', seguido de otra 'a' o una 'b').*

---

## 6. Declaraciones de Entidades
Permiten definir constantes (variables que almacenan un texto) para reutilizarlas en el documento.
**Sintaxis:**
```xml
<!ENTITY nombreEntidad "valorEntidad">
```

---

## 7. Declaraciones de Atributos (`<!ATTLIST>`)
Definen qué atributos puede tener un elemento, de qué tipo deben ser y si son obligatorios o tienen un valor por defecto.

**Sintaxis general:**
```xml
<!ATTLIST nombreElemento nombreAtributo tipoAtributo valorInicialAtributo >
```
*(Nota: Se pueden definir varios atributos en declaraciones separadas o agrupándolos todos dentro de un mismo `<!ATTLIST ... >`).*

### 7.1. Tipos de Atributos Permitidos:
*   **`CDATA`**: Contiene caracteres (texto libre sin restricciones).
*   **`NMTOKEN`**: Solo contiene letras, dígitos, puntos, guiones (`-`), subrayados (`_`) y dos puntos (`:`). **No permite espacios en blanco**.
*   **`NMTOKENS`**: Igual que `NMTOKEN`, pero permite espacios en blanco (sirve para poner una lista de NMTOKENs).
*   **Enumeración `(opcion1 | opcion2)`**: El atributo solo puede tomar uno de los valores listados entre paréntesis y separados por `|`.
    *   *DTD:* `<!ATTLIST ejemplo color (azul | blanco | rojo) #REQUIRED>`
*   **`ID`**: El valor de este atributo funciona como un identificador **único**. No se puede repetir en todo el documento.
*   **`IDREF`**: El valor debe coincidir obligatoriamente con el valor de un atributo `ID` existente en otro elemento (es como una clave foránea).
    *   *XML:* `<libro codigo="L1">...</libro> <prestamo libro="L1">...</prestamo>`
*   **`IDREFS`**: Igual que `IDREF`, pero permite una serie de valores separados por espacios.
    *   *XML:* `<prestamo libro="L1 L2">...</prestamo>`
*   **`ENTITY`**: El valor del atributo debe ser el nombre de una entidad definida previamente.
*   **`ENTITIES`**: Igual que ENTITY, pero para una lista de entidades separadas por espacios.

### 7.2. Valores Iniciales y Obligatoriedad:
Define el comportamiento del atributo si el usuario lo escribe o lo omite en el XML.
*   **`#REQUIRED`**: El atributo es **obligatorio**. No se indica valor por defecto.
*   **`#IMPLIED`**: El atributo es **opcional**. No se indica valor por defecto.
*   **`"valor"` (Predeterminado)**: El atributo tiene un valor por defecto. Si en el XML no se escribe el atributo, el procesador asume este valor automáticamente.
    *   *DTD:* `<!ATTLIST ejemplo color CDATA "verde">`
*   **`#FIXED "valor"`**: El atributo es **fijo**. Si aparece en el XML, obligatoriamente tiene que tener el valor indicado.
    *   *DTD:* `<!ATTLIST ejemplo color CDATA #FIXED "verde">`

## 8. Limitaciones de los esquemas DTD
Aunque DTD fue el estándar inicial, tiene importantes carencias que provocaron la creación de XSD:
1.  **No puede verificar si está bien formado:** DTD es un archivo de texto plano con una sintaxis especial, no es un documento XML en sí mismo.
2.  **Sin tipado de datos:** No se pueden fijar restricciones avanzadas sobre los valores (por ejemplo, obligar a que un contenido sea de tipo `integer`, `date`, delimitar tamaños máximos, etc.).
3.  **No soporta Espacios de Nombres (Namespaces):** Lo que imposibilita mezclar vocabularios con seguridad.
4.  **Restricciones de enumeración limitadas:** Solo permite enumerar valores para los *atributos*, pero no para el contenido de los *elementos*.
5.  **Sin valores por defecto en elementos:** Solo se puede dar un valor por defecto a los *atributos*, no a los *elementos*.
6.  **Control de cardinalidad muy pobre:** Solo permite usar `*`, `+` y `?`. No se puede especificar un número exacto de repeticiones (ej. "este elemento debe aparecer exactamente 5 veces" o "entre 3 y 8 veces").
