Ejercicio 01 Tramitación de pedido online
Enunciado
Diseña el diagrama de actividad del proceso de tramitación de un pedido online desde que el cliente pulsa «comprar» hasta la entrega. Usa swimlanes (Cliente, Almacén, Transportista, Sistema de Pagos) e incluye al menos dos decisiones y un fork/join.
Solución
Swimlanes
Cliente | Sistema de Pagos | Almacén | Transportista
Flujo principal
Confirmar pedido → Procesar pago → [¿Pago OK?] → Verificar stock → [¿Stock OK?] → Preparar pedido → Fork → (Actualizar stock ∥ Registrar bulto) → Join → Transportar → Entregar → Fin
Bifurcación pago fallido
[¿Pago OK? NO] → Notificar error pago → Fin.
Bifurcación sin stock
[¿Stock OK? NO] → Notificar ruptura + Reembolso → Notificar cliente → Fin.
Fork/Join
Después de empaquetar: Almacén actualiza stock y Transportista registra bulto en paralelo; Join antes de Transportar.

Ejercicio 02 Incorporación de nuevo empleado
Enunciado
Elabora el diagrama de actividad del proceso de incorporación de un nuevo empleado. Swimlanes: RRHH, IT, Responsable de Equipo y Empleado. Incluye actividades paralelas con fork/join para entrega de equipo, creación de cuentas y planificación de bienvenida.
Solución
Swimlanes
RRHH | IT | Responsable de Equipo | Empleado
Flujo
Preparar documentación → Firmar contrato → Registrar en sistema → Fork:
Ramas paralelas
A: Preparar equipo → Instalar software → Entregar equipo; B: Crear cuentas → Configurar accesos; C: Planificar bienvenida → Asignar mentor.
Join → continúa
Recibir equipo+credenciales → Presentar al equipo → Completar formación → [¿Superada?] → Activar nómina → Fin.
Ciclo formación
[¿Superada? NO] → volver a Completar formación.

Ejercicio 03 Desarrollo de funcionalidad ágil
Enunciado
Crea el diagrama de actividad del desarrollo de una funcionalidad ágil: desde el user story hasta el despliegue. Swimlanes: Product Owner, Desarrollador, QA y DevOps.
Solución
Swimlanes
Product Owner | Desarrollador | QA | DevOps
Flujo
Escribir User Story → Priorizar → Tomar tarea → Implementar + tests unitarios → Crear Pull Request → Fork:
Ramas paralelas
A: Code review (Desarrollador); B: Tests de integración (QA).
Join → decisión
[¿Review+QA OK?] → NO: Corregir → volver a PR; SÍ: Merge + Pipeline CI.
CI/CD
[¿Pipeline OK?] → NO: Corregir → SÍ: Desplegar en producción → Verificar criterios de aceptación → Fin.

Ejercicio 04 Atención al cliente en call center
Enunciado
Diseña el diagrama de actividad del proceso de atención al cliente en un call center. Incluye gestión de llamadas, consulta de historial, resolución o escalado a especialista y envío de encuesta.
Solución
Swimlanes
Cliente | Agente | Sistema CRM | Especialista
Flujo
Llamar → Atender → Identificar cliente → Consultar historial → Clasificar incidencia
Decisión
[¿Puede resolver el agente?] → SÍ: Resolver problema; NO: Escalar → Atender caso (Especialista).
Registro y cierre
Registrar resolución (CRM) → Confirmar con cliente → Enviar encuesta → Fin.
Notas
Si el cliente no queda satisfecho se puede re-escalar; el CRM recibe eventos de todas las swimlanes.

Ejercicio 05 Aprobación de documento corporativo
Enunciado
Modela el flujo de aprobación de un documento corporativo. Swimlanes: Autor, Revisor, Director y Archivo. El documento puede ser aprobado, devuelto para correcciones o rechazado definitivamente.
Solución
Swimlanes
Autor | Revisor | Director | Archivo
Flujo
Redactar → Enviar a revisión → Revisar → [¿Revisión OK?]
Revisión
Correcciones: Corregir → volver a Revisión; OK: Elevar a Director.
Director
[¿Decisión?] → Aprobado: Firmar → Archivar+Distribuir → Fin.
Director (rechazos)
Devuelto: Corregir (Autor) → volver a Revisión; Rechazado: Notificar rechazo → Fin (sin archivar).

Ejercicio 06 Gestión de incidencias de seguridad informática
Enunciado
Elabora el diagrama de actividad de la gestión de incidencias de seguridad informática. Swimlanes: Usuario, Helpdesk, Equipo de Seguridad y Dirección.
Solución
Swimlanes
Usuario | Helpdesk | Equipo de Seguridad | Dirección
Flujo inicial
Detectar → Reportar → Registrar ticket → Clasificar severidad.
Decisión
[¿Crítica?] → SÍ: Análisis forense + Notificar Dirección; NO: Triaje estándar. → Contener amenaza.
Fork/Join
Erradicar malware ∥ Parchear vulnerabilidad (paralelo) → Join → Restaurar sistemas.
Cierre
Redactar informe → Revisar (Dirección) → Cerrar ticket → Lecciones aprendidas → Fin.

Solución
Swimlanes
RRHH | Responsable de Área | Candidato
Flujo

Descarte
NO: Descartar + Notificar → Fin.
Entrevistas

Oferta y contrato
SÍ: Enviar oferta económica → [¿Acepta?] → Firmar contrato → Iniciar onboarding → Fin.

Ejercicio 08 Facturación y cobro
Enunciado
Crea el diagrama de actividad del proceso de facturación y cobro. Incluye generación de factura, envío, recepción del pago, registro contable y proceso de reclamación por impago.
Solución
Swimlanes
Dep. Comercial | Administración | Cliente | Contabilidad
Flujo
Cerrar servicio → Generar factura → Enviar factura → [¿Factura correcta?]
Error en factura
NO: Reclamar error → Emitir rectificativa → volver a Enviar.
Cobro
SÍ: Esperar vencimiento → [¿Pago a tiempo?] → SÍ: Registrar cobro → Conciliar → Fin.
Impago
NO: Primer recordatorio → [¿Paga?] → NO: Segundo aviso → Escalar a recobro externo → Provisión dudoso cobro → Fin.

Ejercicio 09 Publicación de artículo científico
Enunciado
Elabora el diagrama de actividad de la publicación de un artículo científico. Swimlanes: Autor, Editor, Revisor 1 y Revisor 2.
Solución
Swimlanes
Autor | Editor | Revisor 1 | Revisor 2
Flujo inicial
Redactar → Enviar → Verificar formato/alcance → [¿En alcance?] → NO: Rechazar → Fin.
Revisión paralela
SÍ: Fork → (Revisor 1 revisa ∥ Revisor 2 revisa) → Join → Consolidar dictámenes.
Decisión editorial
[Rechazo] → Notificar → Fin; [Rev. mayor] → Autor revisa (60 días) → volver a Fork; [Rev. menor] → Autor corrige → Verificar correcciones; [Aceptado] → Editar y publicar → Fin.

Ejercicio 10 Mantenimiento preventivo de flota
Enunciado
Diseña el diagrama de actividad del mantenimiento preventivo de una flota. Swimlanes: Operador, Mecánico, Almacén de Repuestos y Responsable de Flota. Usa fork/join para actividades paralelas.
Solución
Swimlanes
Operador | Mecánico | Almacén de Repuestos | Responsable de Flota
Flujo
Generar plan → Entregar vehículo → Inspección inicial → Fork:
Paralelo
A: Cambio aceite+filtros (Mecánico + Almacén); B: Revisar frenos+neumáticos; C: Actualizar firmware ECU.
Join → decisión
[¿Anomalía?] → SÍ: Presupuestar → [¿Aprobado?] → SÍ: Reparar; NO: Registrar y diferir.
Cierre
Prueba de carretera → Registrar mantenimiento → Recoger vehículo → Actualizar plan → Fin.
